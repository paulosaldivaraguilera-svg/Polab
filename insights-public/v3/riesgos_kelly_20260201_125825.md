# Gestión de Riesgos - Trading Polymarket

## Tríada de Riesgos

| Riesgo | Descripción | Mitigación |
|--------|-------------|------------|
| **Ejecución** | Latencia, slippage | L2 Auth, WebSocket |
| **Modelo** | False signals | Paper trading 2+ semanas |
| **Capital** | Ruina | Kelly fraccional + Stop |

## Criterio de Kelly

### Fórmula
```
f* = (p(b+1) - 1) / b

Donde:
p = probabilidad de victoria (del modelo)
b = odds - 1
```

### Aplicación práctica
```
Kelly óptimo = 25%
Position Size = Kelly × Capital × 0.25
```

### Ejemplo
- Win rate: 55%
- Odds: 2.0
- Kelly = (0.55 × 3 - 1) / 2 = 0.325 (32.5%)
- Fraccional = 0.325 × 0.25 = 8.1%
- Position = 8.1% × $1000 = $81

## Kill Switch Automático

### Condiciones de parada

| Check | Threshold | Acción |
|-------|-----------|--------|
| Drawdown diario | > 15% | Stop bot |
| Pérdida diaria | > $50 | Stop bot |
| Errores API | > 10/min | Pause 15min |
| Temperatura Pi | > 75°C | Warning |

## Reglas de Oro

1. **Paper primero**: 2 semanas mínimo
2. **Kelly fraccional**: Nunca 1x
3. **Kill switch**: SIEMPRE activo
4. **L2 Auth**: Nunca private key en código
5. **Logs**: Todo queda registrado
