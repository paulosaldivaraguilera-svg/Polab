# Optimización para Chile

## Desafío de Latencia

| Destino | Latencia Típica |
|---------|-----------------|
| US East (AWS) | 110-140ms |
| US West | 200-250ms |
| Europa | 300-350ms |
| Sao Paulo | 80-100ms |

## Estrategias de Mitigación

### 1. DNS y Resolver
```bash
# Usar Cloudflare DNS
echo "nameserver 1.1.1.1" | sudo tee /etc/resolv.conf
```

### 2. Protocolo de Conexión
- **WebSocket (WSS)** sobre HTTP
- Mantener conexión persistente
- Eliminar handshake TCP/TLS repetido

### 3. VPN/Proxy (Opcional)
- VPS en Miami/Sao Paulo como relay
- Mejora enrutamiento pero añade latencia extra
- Usar solo si ruta directa es inestable

### 4. Hardware
- Raspberry Pi 5 (mejor CPU para crypto ops)
- NVMe boot (reduce I/O latency)
- Ethernet cableado (no WiFi)

## Configuración Red Chile

```bash
# Priorizar traffic a Polymarket
sudo tee /etc/sysctl.conf << 'EOF'
net.core.rmem_max = 134217728
net.core.wmem_max = 134217728
net.ipv4.tcp_rmem = 4096 87380 134217728
net.ipv4.tcp_wmem = 4096 65536 134217728
